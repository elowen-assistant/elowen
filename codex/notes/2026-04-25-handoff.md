## Delivered

- `elowen-edge` now requires TOML runtime startup through `elowen-edge run --config <path>`.
- Legacy env files are supported only through one-time import: `elowen-edge config import-env --env-file <path> --config <path>`.
- Edge trust material now lives in separate local secret files referenced by TOML, with Unix permission checks for secret-file mode.
- The edge binary includes a TUI for dashboard health, first-run checklist, config validation, service controls, and diagnostics.
- Runtime status is written to `[runtime].state_dir/status.json`.
- Windows helper scripts now launch the TOML runtime path through a hidden scheduled-task launcher so the background edge does not leave an operator terminal open.
- The TUI now uses arrow-key tab navigation, ignores key-release/repeat events, displays registration timestamps in the local timezone, explains first-run/service states, and can auto-discover a runnable Codex command.
- Windows release packaging now includes an unsigned Inno Setup `ElowenEdgeSetup.exe` installer that installs the binary, TOML config, secret files, scheduled task, TUI shortcuts, and uninstall entry.
- The self-contained `Install-ElowenEdge.ps1` bootstrap installer remains available as a lower-level debug fallback.
- Linux systemd and Windows Task Scheduler service helpers are available through the TUI/service module.
- `elowen-edge` now has a GitHub Actions workflow for Windows and Linux executable artifacts.
- Local Compose edge startup now uses `env/edge.compose.toml`.
